# 戦争OSINT代替: AI/Web3/収益化 10分実写動画生成 自動化仕様

## 目的

戦争OSINTの10分長尺動画枠の代わりに、以下のテーマで伸びやすい長尺動画を自動生成する。

- Web3
- クリプト
- Codex
- Claude Code
- バイブコーディング
- YouTube / X / Threads / Instagram / TikTok
- SNS収益化
- AIで稼ぐ、OSSで稼ぐ、個人開発で稼ぐ

これは短尺テストではなく、戦争OSINT枠の置き換えなので、基本は10分程度の長尺動画として作る。

## 重要な方針

- ERNIEなどで生成した静止画スライド中心の動画にはしない。
- 戦争OSINT動画のように、実写・画面録画・資料映像を使った動画にする。
- 台本と無関係な汎用実写素材を当てない。
- 元ネタとなるYouTube動画やPDF資料の内容に基づいて、映像も台本も作る。
- 失敗したときに、関係ない素材や薄い台本でごまかして動画を作らない。
- 品質が低い場合は動画生成を中止し、エラー理由を残す。

## 入力素材

優先順位は以下。

1. 海外でバズっているYouTube動画
2. 収益化、AI活用、Codex、Claude Code、Web3などに関するPDF資料
3. ブログ記事や技術記事

重要: 「1時間以上のYouTube動画を10分にまとめる」は有効な方法の1つだが、絶対条件ではない。
本質は、題材がAI/Web3/Codex/Claude Code/SNS収益化系で、元素材の内容を分析し、実写・画面・資料の情報量を持った10分考察動画へ再構成すること。

YouTube動画を使う場合は、以下を取得する。

- タイトル
- 概要欄
- 字幕または自動字幕
- 再生数などの参考情報
- 動画本体からの短い抜粋映像

PDFを使う場合は、以下を取得する。

- PDF本文
- ページ画像
- 図表、画面キャプチャ、スライド

## 映像の作り方

- 元YouTube動画から複数箇所を抜粋して使う。
- PDFの場合はページ画像を使い、ズームやパンで動きをつける。
- 映像は台本内容と関係があるものにする。
- 無関係なWikimediaや汎用オフィス映像に逃げない。
- 元動画の音声は基本的に使わず、Kurage側の日本語ナレーションを乗せる。
- ただし、画面上で元動画や資料を見ていることがわかる構成にする。

## 台本の作り方

台本は、元YouTube動画やPDFの内容を忠実に整理する。

必ず入れる要素:

- 何がバズっているのか
- どこに収益化のポイントがあるのか
- 具体的な手順
- 必要なツールやOSS
- 再現条件
- リスク
- Kurage / kagentreach / kdeck / kargov / ksnsposter などに応用できる点

禁止事項:

- 同じ内容の繰り返し
- 当たり障りない一般論だけの台本
- URLをアルファベットで朗読すること
- 英語タイトルや英文を長くそのまま読むこと
- 「素材のライセンス」「この動画では」など制作上の注意事項をナレーションに入れること
- 戦争、地政学、OSINTの話に戻ること

## 生成時間

- 本番想定は10分程度。
- 戦争OSINT枠の代替なので、短尺動画ではなく10分長尺として成立させる。
- 1時間以上の元YouTubeを10分に圧縮するのは有力な方法だが、必須条件ではない。
- PDFやブログを題材にする場合も、10分程度の考察動画に再構成する。
- 2分などの短尺で作らない。
- テストで短尺にする場合は、明示的に「短尺テスト」として扱い、本番代替の成功とは言わない。

## 出力先

まずはYouTubeへ直接投稿しない。

1. Kurageに動画として登録する。
2. `kuragev.php` で再生確認できるようにする。
3. 内容、映像、台本、音声を確認する。
4. 良い動画ができたら、YouTube投稿の自動化へ切り替える。

## 自動化の責務

kagentreach:

- 海外で伸びているYouTube動画、PDF、記事を探す。
- 参照元URL、タイトル、概要、字幕、本文を取得する。
- 生成対象の候補を管理する。

Kurage / kurage_web:

- 10分動画を生成する。
- Kurage動画として登録する。
- `kuragev.php` で確認できるようにする。

kdeck / rqdb4ai:

- workerジョブとして実行管理する。
- 成功、失敗、途中経過を確認できるようにする。

## 品質ゲート

以下の場合は動画を作らず失敗にする。

- 元動画やPDFの内容が取得できない。
- 字幕、概要、本文が不足していて台本が薄くなる。
- 台本が繰り返しになっている。
- 台本が元素材と関係ない。
- 映像が台本と関係ない。
- 10分動画として成立していない。
- 音声や字幕が壊れている。

## 今回の反省点

- 2分テストを本命のように進めたのは間違い。
- 戦争OSINT代替なので、最初から10分動画として扱うべきだった。
- 「実写がいい」は、汎用実写素材ではなく、元YouTube動画の抜粋やPDF資料画像を使うという意味。
- 成功確認前に作業を進めすぎない。
- 短尺テストをする場合は、本番仕様と混同しない。
